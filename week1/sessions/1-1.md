## Block 3 — 검증 + 재수정 (40분)

### 검증

- for-loop 이 NumPy 보다 **수십~수백 배 (Apple Silicon 은 수만 배)** 느려야 정상
- NumPy 결과가 GFLOPs 기준 **10~수천 GFLOPs** 범위면 합리적
- 두 결과 행렬이 `np.allclose(..., atol=1e-3)` 일치

### 재수정 — 흔한 추가 지시

- warmup 1회 누락 → 첫 실행이 비정상적으로 느림 → "측정 전 warmup 1회 추가"
- 결과 검증 코드 누락 → "두 방식의 결과를 `np.allclose` 로 비교"
- 1000×1000 for-loop 이 너무 오래 → "N=200 으로 축소"
- 에이전트가 GPU 코드 작성 → "CPU 에서만, NumPy 만 사용"

### 예상 결과 (참고)

```
Loop  : 1.4 s     (≈ 0.01 GFLOPs)
NumPy : 0.02 ms   (≈ 900 GFLOPs)
Speedup: ~80,000x  (Apple M4 Max)  또는 ~700x (Intel/AMD typical)
allclose: True
```

자릿수만 비슷하면 OK. 절대값은 기기마다 다름.

## 흔한 막힘 포인트

| 증상 | 원인 | 해결 |
| --- | --- | --- |
| `ModuleNotFoundError: numpy` | 가상환경 비활성 | `source .venv/bin/activate` |
| for-loop 이 5분 넘게 안 끝남 | 1000×1000 은 순수 Python 에 너무 큼 | N=200 또는 100 으로 축소 |
| 첫 실행만 비정상적으로 느림 | warmup 효과 | 측정 전 1회 warmup |
| 두 결과 미세하게 다름 | float 누적 오차 | `np.allclose(atol=1e-3)` |
| 에이전트가 GPU 코드 작성 | 의도 불명확 | "CPU 만, NumPy 만" 명시 |
