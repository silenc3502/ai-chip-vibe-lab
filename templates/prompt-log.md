# 프롬프트 로그 템플릿

> 이 과정의 모든 실습 산출물에는 **프롬프트 로그**를 첨부합니다. 단순 복붙을 막고, "왜 이렇게 지시했는가"를 학습 기록으로 남기는 것이 목적입니다.

---

## 작성 규칙

각 실습 단위(보통 30분 ~ 1시간 분량)마다 아래 5단계를 한 블록으로 기록합니다.

### 1. 의도 (Intent)
무엇을, 왜 하려고 하는가? 한두 문장.

### 2. 프롬프트 (Prompt)
실제 에이전트에게 입력한 프롬프트 원문. 여러 번 시도했다면 모두 기록.

### 3. 결과 (Result)
에이전트가 만든 코드 / 답변의 핵심. 전체를 붙이지 말고 변경된 파일·요약만.

### 4. 검증 (Verification)
- 실행했는가? 어떤 결과가 나왔는가?
- 기대와 다른 점이 있었는가?
- 수치는 합리적인가?

### 5. 재수정 (Revision)
검증 결과에 따라 어떤 추가 지시를 했는가, 혹은 직접 코드를 수정했는가.
*"한 번에 성공"인 경우에도 그 사실을 기록.*

---

## 예시

### Block 1 — NumPy MatMul 베이스라인 측정

**1. 의도**
> 1024×1024 float32 행렬곱 시간을 측정하는 베이스라인 스크립트를 만들고 싶다. 이후 GPU 모델과 비교할 기준점이다.

**2. 프롬프트**
> "Python 스크립트로 1024x1024 float32 행렬 두 개를 만들고 NumPy로 곱한 시간을 5회 평균으로 측정해줘. 결과는 GFLOPs 단위로 출력."

**3. 결과**
- 새 파일: `week1/lab/matmul_baseline.py`
- `time.perf_counter`로 측정, GFLOPs 계산식 포함

**4. 검증**
- 실행 결과: 평균 92 GFLOPs (M2 Mac, NumPy 1.26)
- 기대 범위(50~150 GFLOPs) 내. OK.

**5. 재수정**
- warmup 1회 누락이 의심되어 "측정 전에 warmup 1회를 추가" 추가 지시.
- 결과 95 GFLOPs로 안정화.
