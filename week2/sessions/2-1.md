## Block 1 — MatMul 분해 + 6 loop ordering (40분)

### 세 가지 분해

`C[i,j] = sum_k A[i,k] * B[k,j]` 를 어느 차원이 outer 냐에 따라:

- **Inner product** (k 가 inner): `for i: for j: for k: ...`
- **Outer product** (k 가 outer): `for k: for i: for j: ...`
- **Gustavson** (i 가 outer, k middle): `for i: for k: for j: ...`

### 6 ordering — inner-most 가 캐시 친화성 결정

| Order | 가장 안쪽 접근 | C-contiguous 친화성 |
| --- | --- | --- |
| ijk | A[i,k], B[k,j] | A row, B col(strided) |
| ikj | A[i,k], B[k,j], C[i,j] | A 고정, B/C row ✓ |
| jik | A[i,k], B[k,j] | A row, B col |
| jki | A[i,k], B[k,j], C[i,j] | A/C col(strided) |
| kij | A[i,k], B[k,j], C[i,j] | A 고정, B/C row ✓ |
| kji | A[i,k], B[k,j], C[i,j] | A/C col(strided) |

> 직관: **`ikj`, `kij` 가 빠를 후보** (j 가 inner-most → row 순회). 실험에서 확인.

### 토의 (5분)

1. `ikj` 와 `kij` 가 비슷한 이유 (둘 다 j inner-most)
2. *연산량 같은데 시간 다른* 이유를 1-2 교훈으로 한 줄

## Block 2 — 실험: 6 ordering 시간 비교 (40분)

### 학생 과제 (바이브 코딩)

> N=120 정사각 float32 NumPy 배열에 대해 6 ordering MatMul 함수 6개를 작성 (NumPy ndarray 인덱싱 사용, 벡터 연산 X). 시간 측정 + `np.allclose` 검증.

### 검증 기준

- 6 결과 모두 `np.allclose(..., A @ B, atol=1e-3)` True
- 일반: 가장 빠른/느린 비율 **1.5~5x**
- **Apple Silicon 예외**: 비율 **1.05x 미만** 가능 — 인터프리터 오버헤드(~30ns/access) 가 cache miss 비용(~5ns) 압도. 학습 목표는 *개념*, 측정 수치 X
- 명확한 차이 보려면 Numba `@njit` (선택 도전, +5-20x)

## Block 3 — Dataflow + Stationary 분류 (40분)

### "Stationary" 정의

> Inner-most loop 에서 **인덱스가 변하지 않는** 변수

| Order | inner-most | stationary | 분류 |
| --- | --- | --- | --- |
| ijk | k | C | **Output Stationary** |
| jik | k | C | **Output Stationary** |
| ikj | j | A | **Input Stationary** |
| kij | j | A | **Input Stationary** |
| jki | i | B | **Weight Stationary** |
| kji | i | B | **Weight Stationary** |

### Dataflow 다이어그램 작성

각 stationary 에 대해 4 요소 표현 (손그림 OK):

1. PE 에 *고정* 된 데이터 (stationary)
2. *흘러 들어오는* 데이터
3. *흘러 나가는* 출력
4. 한 사이클의 multiply-accumulate

> 손그림 사진을 `figures/` 에 저장.

### 핵심 통찰

- **6 ordering = 3 stationary × 2 외부 순서**
- 하드웨어는 *하나를 골라 실리콘에 못박는* 것 — 2-2 에서 weight stationary 가 실리콘이 되는 사례
