## Part A. 칩 카탈로그 강의 (60분)

### 분석 후보 칩

학생은 다음 중 **1종을 선택**합니다 (강사가 사전에 분배해도 OK).

| 칩 | 회사 | Defining feature (1줄) |
| --- | --- | --- |
| Tenstorrent Wormhole / Grayskull | Tenstorrent | RISC-V 기반 그래프 컴퓨팅, packet-based 통신 |
| Groq LPU | Groq | 완전 결정론적 dataflow, 컴파일러가 모든 사이클을 스케줄링 |
| Cerebras WSE-2/3 | Cerebras | Wafer-scale (전체 웨이퍼 한 칩), on-chip 40GB SRAM |
| Graphcore IPU | Graphcore | 큰 on-chip SRAM (~900MB), BSP 동기화 |
| SambaNova SN30/40L | SambaNova | Reconfigurable dataflow, RDU |
| Untether AI Boqueria | Untether | At-memory compute, SRAM 옆에서 직접 연산 |
| Mythic M1076 | Mythic | Analog in-memory compute (flash transistor) |
| NVIDIA Hopper Transformer Engine | NVIDIA | FP8 + 동적 dtype switching for transformers |
| Intel Gaudi 3 | Intel | RDMA 기반 large-scale interconnect |

> 같은 칩을 여러 학생이 골라도 OK — 각자 다른 백서에서 출발하면 분석이 자연스럽게 달라짐.

### 강의 핵심

- *모든 칩은 "어디에 베팅했는가" 의 결과* — 하나의 axis (정밀도 / 통신 / 메모리 / 결정론) 에 칩의 정체성이 응축됨.
- 백서를 읽을 때 찾을 것:
  1. peak TFLOPS / TOPS (어느 dtype에서?)
  2. 메모리 BW (HBM? on-chip SRAM 크기?)
  3. compute 단위 — systolic? SIMT? dataflow graph? 무엇?
  4. 통신 — 어떻게 멀티칩 확장?
  5. *대표적 application benchmark* — 회사가 자랑하는 워크로드

### 자료 출처

- 회사 공식 백서 (위키피디아 → 공식 사이트)
- HotChips, IEEE ISSCC, MICRO 학회 발표
- SemiAnalysis 같은 분석 사이트 (단 의견 ≠ 사실)
- *명목값(spec)* 과 *실측 벤치마크*를 구분하라

## 흔한 막힘 포인트

| 증상 | 원인 | 해결 |
| --- | --- | --- |
| 백서가 영어라 어려움 | 자연스러움. 정공법 | 에이전트에게 "이 백서의 architecture 섹션을 한국어로 요약" 요청, 단 *원문 인용*은 본인이 확인 |
| Defining feature가 "AI에 빠르다" 수준 | 다른 칩도 다 그 말을 함 | "이 칩만 갖는 결정"을 강제로 한 단어로 — 정밀도? 메모리 위치? 결정론? 그래프? |
| roofline 적용에 reuse_factor를 모름 | 백서에 명시 안 됨 | array dimension에서 추정 또는 합리적 가정 + 가정을 보고서에 명시 |
| 강점이 3개 모두 비슷한 말 | 같은 한 가지 특징의 다른 표현 | 1개는 정밀도/dtype, 1개는 데이터 이동, 1개는 통신 — *서로 다른 axis*에서 뽑기 |
