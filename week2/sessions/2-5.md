## Block 1 — 칩 카탈로그 강의 + 칩 선택 (40분)

### 분석 후보 (1종 선택)

| 칩 | 회사 | Defining feature |
| --- | --- | --- |
| Tenstorrent Wormhole | Tenstorrent | RISC-V 그래프 컴퓨팅, packet-based 통신 |
| Groq LPU | Groq | 결정론적 dataflow, 컴파일러 사이클 스케줄링 |
| Cerebras WSE-2/3 | Cerebras | Wafer-scale, 40GB on-chip SRAM |
| Graphcore IPU | Graphcore | 큰 on-chip SRAM (~900MB), BSP 동기화 |
| SambaNova SN30/40L | SambaNova | Reconfigurable dataflow, RDU |
| Untether AI Boqueria | Untether | At-memory compute |
| Mythic M1076 | Mythic | Analog in-memory compute (flash) |
| Hopper Transformer Engine | NVIDIA | FP8 + 동적 dtype switching |
| Intel Gaudi 3 | Intel | RDMA 기반 large-scale interconnect |

> 같은 칩 여러 학생 OK (백서 다르면 분석 자연스레 달라짐).

### 백서 읽을 때 찾을 5가지

1. peak TFLOPS/TOPS (어느 dtype?)
2. 메모리 BW (HBM? on-chip SRAM 크기?)
3. compute 단위 (systolic? SIMT? dataflow graph?)
4. 통신 (멀티칩 확장 방법?)
5. 대표 application benchmark (회사가 자랑하는 워크로드)

### 2. Dataflow 분류

- WS / OS / IS / RS / 기타 (또는 *완전히 새로운 클래스*)
- 근거 (백서 인용 또는 추정)

### 3. Roofline 위치

- Week 2-4 의 5 워크로드에 본인 칩 예측 시간 계산
- 표에 추가 (CPU/GPU/NPU/TPU + 본인 칩)
- 가장 강한 / 약한 워크로드 1개씩

### 4. Defining feature

- *이 칩만 갖는 한 가지 결정* 은? — 정밀도? 메모리 위치? 결정론? 그래프?
- Trade-off (얻은 것 / 잃은 것)

## Block 3 — 보고서 작성 (40분)

### 템플릿 (`chip_analysis_<칩>.md`)

```markdown
# <칩 이름> 아키텍처 분석

## 1. 한 줄 요약 — 정체성을 한 문장으로
## 2. 기본 스펙 (표)
## 3. 블록 다이어그램 (직접 또는 백서 발췌, 출처)
## 4. Dataflow 분류
## 5. Roofline 비교 (Week 2-4 표 + 본인 칩)
## 6. Defining feature + Trade-off
## 7. 강점 (3 bullet, *서로 다른 axis*)
## 8. 약점 (3 bullet)
## 9. 적합한 워크로드 (3, 구체적으로)
## 10. 부적합한 워크로드 (1-2)
## 11. 한 줄 평 — "같은 작업을 X 도 할 수 있다. 하지만 이 칩은 _____ 때문에 더 ___ 하다."
## 출처
```

### 분량

- A4 1-2 page (markdown)
- 표 1개 이상, 그림 1개 이상

## 흔한 막힘 포인트

| 증상 | 원인 | 해결 |
| --- | --- | --- |
| 백서가 영어라 어려움 | 자연스러움 | 에이전트에게 *"architecture 섹션 한국어 요약"* — 단 *원문 인용*은 본인 확인 |
| Defining feature 가 *"AI 에 빠르다"* 수준 | 다른 칩도 같은 말 | *"이 칩만 갖는 결정"* 한 단어로 강제 |
| roofline 의 reuse_factor 백서에 없음 | 추정 필요 | array dim 에서 추정 + *가정을 명시* |
| 강점 3개가 비슷한 말 | 같은 특징의 다른 표현 | 1개는 정밀도, 1개는 데이터 이동, 1개는 통신 — *서로 다른 axis* |
