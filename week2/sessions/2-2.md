## Block 1 — TPU 역사 + 설계 동기 (40분)

### 왜 Google 이 TPU 를?

- 2013: 음성검색에 DNN → 데이터센터 2배 필요 추산
- 2015: TPU v1 (15개월 만에). INT8, 256×256 systolic = 65,536 MAC

### 세대별

| 버전 | 출시 | 정밀도 | systolic | 용도 |
| --- | --- | --- | --- | --- |
| v1 | 2015 | INT8 | 256×256 | 추론 |
| v2/3 | 2017/18 | BF16 | 128×128 ×2 | 학습 |
| v4 | 2022 | BF16 | 128×128 ×4 | OCS optical |
| v5 | 2023 | BF16/INT8 | 확장 | 추론/학습 분기 |

### Architectural commitment

- *"MatMul 이 99% 워크로드"* 라는 가정에 *모든 다이 면적* 베팅
- Conv, Attention 도 결국 MatMul 로 환원. 그 외 op 은 host

## Block 2 — Systolic Array 메커니즘 (40분)

### 한 PE 의 동작

```
weight register : 가중치 (고정)
multiplier      : 입력 × 가중치
adder           : 위에서 내려온 partial sum + multiplier 결과
```

매 사이클: 입력 좌→우 흐름, partial sum 위→아래 흐름

### 4×4 systolic Weight Stationary (강사 시연)

`C = A · B`, A/B/C 모두 4×4:
- B 의 16 element 를 4×4 PE 에 미리 로드 (PE[i,j].weight = B[i,j])
- A 의 행을 *staggered* 입력 (i 번째 행이 i 사이클 지연)
- C 결과는 *아래쪽 가장자리* 에서 나옴
- ~11 사이클 (M+K+N-1)

### 왜 빠른가 — Reuse

| 대상 | reuse 횟수 (4×4 systolic, 4×4 MatMul) |
| --- | --- |
| Weight | 4 (각 weight 가 A 의 4개 행에 곱) |
| Input | 4 (각 input 이 B 의 4개 열과 곱) |
| Partial sum | 0 메모리 trip (PE 내부 누적) |

→ 256×256 array (TPU v1) 이면 매 사이클 65,536 MAC 동시 실행.

## Block 3 — 실험: 4×4 Systolic 시뮬레이터 (40분)

### 학생 과제 (바이브 코딩)

> 4×4 systolic array 시뮬레이터 Python 클래스. Weight Stationary. PE 별 weight + partial_sum 상태. `step()` 메서드로 한 사이클 진행, 매 step 마다 PE 격자 상태 출력. 4×4 A, B 로 C = A·B 계산, NumPy 와 `np.allclose` 검증.

### 시뮬레이터 사양

- `SystolicArray(rows, cols)` — 4×4
- 메서드: `load_weights(B)`, `feed_input(A)`, `step()`, `collect_output()`
- 매 사이클 PE 상태 표 출력

### 검증 기준

- 결과가 `A @ B` 와 `np.allclose`
- 총 사이클 ≈ M + K + N - 1 (4×4 면 11 사이클)
- PE 상태 표 출력 가능 (디버깅용)

### 출력 매핑 공식

```
C[m, n] 은 cycle (m + K + n) 에 bottom row 에서 emerge
```

(reference 솔루션의 trace 참조)

### 추가 도전 (시간 남으면)

- 8×8 일반화
- INT8 정밀도

### Week 3 RTL 미리보기

본 회차에서 작성한 Python systolic 의 단일 PE 동작은 다음 Verilog 와 *정확히 같습니다*:

```verilog
// 3-2 에서 학생이 작성할 단일 MAC (Verilog snippet)
module mac (
    input  wire                clk,
    input  wire                rst,
    input  wire signed  [7:0]  in_data,    // weight stationary 의 input
    input  wire signed  [7:0]  weight,     // PE 에 stationary 한 weight
    output reg  signed  [31:0] acc         // partial sum 누적
);
    always @(posedge clk) begin
        if (rst) acc <= 32'sd0;
        else     acc <= acc + (in_data * weight);   // ← Python systolic 의 1 step
    end
endmodule
```

차이는 *언어* 가 아니라 *대상*:
- **Python**: 분석용. 디버깅 쉬움, 사이클 추적 가능, 수치 ground truth
- **Verilog**: 구현용. 합성 가능, 실제 칩, FPGA 동작 가능

3-2 에서 **본인이 직접 작성** 후 **cocotb 로 Python ↔ Verilog 결과 일치 검증**.
